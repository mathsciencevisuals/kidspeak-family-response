# KidSpeak Roadmap

KidSpeak is an India-first family behavioural communication coach. The product focuses on communication patterns, emotional signals, coaching opportunities, and professional review where needed. It does not claim medical diagnosis for a parent or child.

## Product Expansion

New product name option: **KidSpeak: India-first AI family communication coach.**

Sprint 0 expands the Family Response Intelligence prototype into a low-cost MVP with:

- Audio upload from mobile phone recordings.
- Transcript upload and paste flows to avoid transcription cost.
- Regional language support for English India, Hindi, Telugu, and Tamil.
- Session history and progress review.
- Longitudinal tracking for escalation rate, parent validation improvement, child regulation improvement, repair score trend, trigger frequency, and calmness trend.
- Competitive moat screen for voice NLP, conversation graph, parent + child + therapist views, and India-first localisation.
- Future live coaching screen marked experimental, disabled by default, simulation-first, consent-first, and rule-based before AI.

Transcript upload is the low-cost path: uploaded transcripts are normalized with rule-based parsing, converted to `ConversationTurn` records, marked `transcribed`, and skip transcription. AI speaker inference is not used by default.

Run the API locally with:

```bash
npm run api:dev
```

Storage uses a provider toggle in `src/repositories/storageAdapter.ts`.

- `STORAGE_PROVIDER=redis`: use GCP Memorystore-compatible Redis when `REDIS_URL` is configured, otherwise fall back to memory.
- `STORAGE_PROVIDER=memory`: force in-memory local development.
- `STORAGE_PROVIDER=firebase`: reserved switch for Firebase/Firestore. The adapter currently fails clearly until Firebase credentials and SDK mapping are added.

The API handlers call `Sprint1Repository`, so switching storage providers should not require changing route code.

Cost note: Redis is useful for low-latency session state and compact MVP records, but managed Redis can be an always-on cost. Firebase/Firestore can be switched in later if document pricing and operational simplicity fit better. Keep transcript turns, graph nodes, recommendations, transcript uploads, and session metrics outside the parent session object. Do not use BigQuery in Sprint 1.

Audio upload defaults are configured by environment:

- `AUDIO_MAX_DURATION_SECONDS`, default `300`
- `AUDIO_MAX_FILE_SIZE_BYTES`, default `26214400`
- `AUDIO_RETENTION_DAYS`, default `1`
- `AUDIO_BUCKET_NAME`, default local placeholder bucket

Use Cloud Storage lifecycle deletion for raw audio. Delete raw audio after analysis by default or after `AUDIO_RETENTION_DAYS`.

## Sprint 3 Regional Language Support

Initial language codes:

- English India: `en-IN`
- Hindi India: `hi-IN`
- Telugu India: `te-IN`
- Tamil India: `ta-IN`

Language support is split by use:

- Conversation language is stored on `ConversationSession` and passed as `languageCode` to the transcription provider.
- Uploaded transcripts store transcript language and skip transcription.
- UI language is stored separately from transcript language.
- Recommendation language controls coaching output. If full translation is not ready, output can be bilingual: English plus selected language.

Static localisation assets:

- UI dictionaries: `src/localisation/languages.ts`
- Prompt localisation config: `LanguageConfig` with `languageCode`, `displayName`, `parentTerm`, `childTerm`, `culturallySafeExamples`, and `scriptStyle`
- Transcription provider interface: `src/services/transcriptionProvider.ts`

Low-cost localisation rules:

- Do not translate every page with AI on every load.
- Use static dictionaries for UI labels.
- Use AI only for final coaching script translation if required.
- Cache translated scripts.
- Keep coaching simple and culturally sensitive: respectful communication, emotional regulation, safety, and boundaries.

## Sprint 4 Multilingual NLP

The rule-based NLP engine supports `en-IN`, `hi-IN`, `te-IN`, and `ta-IN` transcripts with multilingual phrase dictionaries for parent escalation phrases and child reaction phrases.

Analysis behavior:

- Preserve original transcript text for UI display.
- Store optional English meaning/translation for analysis only.
- Graph nodes include original utterance, translated meaning when available, detected pattern, coaching recommendation, original language, recommendation language, and analysis confidence.
- Confidence values are `high`, `medium`, and `low`.
- Low-confidence turns are marked `Needs human review`; the engine does not force a conclusion for cultural phrases it cannot confidently match.
- Therapist review shows confidence and original language.

Endpoint:

- `POST /api/sessions/:id/analysis/run`

The endpoint uses stored `ConversationTurn` records and does not call expensive AI inference by default.

## Sprint 5 Parent Coaching

Route:

- `/sessions/:id/parent`

Purpose: help parents see where their response escalated the issue and how to respond better, without using diagnosis or mental-health scoring language.

UI components:

- `ParentPatternCard`
- `ParentCoachingScoreCard`
- `PhraseComparisonCard`
- `ParentScriptBuilder`
- `PracticePlanCard`
- `ProfessionalReviewBanner`

API:

- `GET /api/sessions/:id/parent-analysis`
- `POST /api/sessions/:id/parent-practice-plan`

Parent scripts use the rule-based formula first:

`Observe -> Validate -> Boundary -> Small Next Step`

Gemini should only be called if the parent clicks `Generate personalized script`. Generated scripts are cacheable by session and prompt version; when `STORAGE_PROVIDER=firebase` is wired, this cache can live in Firestore to avoid repeated AI cost.

Safety boundary: if language suggests severe aggression, intimidation, abuse, self-harm threats, or violence, the module recommends professional review and marks the case high or critical instead of showing normal coaching alone.

## Sprint 8 Safety Risk Classifier

Route:

- `/sessions/:id/safety`

API:

- `POST /api/sessions/:id/risk-assessment`
- `GET /api/sessions/:id/risk-assessment`

The safety classifier accepts `ConversationTurn[]` and returns a `RiskAssessment` with risk level, risk categories, detected phrases, explanation, recommended action, and flags for `blockNormalCoaching` and `requireProfessionalReview`.

It runs a rule-based safety pre-check before LLM analysis. Optional Gemini safety analysis is represented as an opt-in path only when uncertainty is high; it is not called by default. If high or critical risk is detected, normal coaching is blocked as the primary result, the app shows a `SafetyReviewBanner`, recommends immediate adult or professional support, allows therapist or psychologist review when consented, stores the assessment securely, and does not gamify the event.

Language rule: do not say “AI confirms abuse” or “AI confirms self-harm.” Use: “This conversation contains concerning language that may require immediate adult or professional attention.”

## Sprint 9 Privacy, Consent, And Retention

Routes:

- `/settings/privacy`
- `/settings/consent`
- `/settings/data-retention`

API:

- `GET /api/privacy/consents?familyId=...`
- `POST /api/privacy/consents`
- `DELETE /api/privacy/consents/:id`
- `POST /api/privacy/delete-session`
- `POST /api/privacy/export-data`
- `GET /api/audit-logs?familyId=...`

Controls:

- Parent can grant or revoke recording consent, therapist sharing consent, data retention consent, and research opt-in.
- Child-friendly notice: “We record only to help understand conversations and practise better responses.”
- Audio retention options: delete raw audio immediately after analysis, keep for 7 days, or keep for 30 days. MVP default is delete after analysis or 7 days maximum.
- Transcript retention options: keep transcript, delete transcript but keep summary, or delete entire session.
- Parent export includes session summaries, transcript, recommendations, and therapist notes visible to parent.
- Parent deletion supports one session, child profile, or all family data.
- Audit log tracks consent granted, consent revoked, audio uploaded, analysis run, therapist viewed, data exported, and data deleted.

GCS lifecycle support is documented in `scripts/apply-gcs-audio-lifecycle.sh`. It writes a placeholder lifecycle rule that deletes `raw-audio/` objects after `AUDIO_RETENTION_DAYS`.

## Longitudinal Intelligence Engine

The deterministic engine in `src/services/longitudinalIntelligence.ts` analyses `SessionMetric[]` for a family and child and returns `TrendInsight[]` records with:

- improvement, concern, repeated trigger, parent growth, child growth, and therapist review insight types
- previous/current values, percent change, explanation, recommended next step, and confidence
- metrics for escalation rate, repair score, parent validation, criticism/threat frequency, child shutdown, clarity, self-regulation, listening balance, and top triggers

API:

- `GET /api/history/trends?familyId=...&childId=...`
- `POST /api/history/generate-trend-snapshot`
- `GET /api/children/:id/trend-insights?familyId=...`

The engine uses deterministic calculations from stored Firestore session metrics and does not call AI by default. Deeper family insight remains a cached, user-triggered Gemini option exposed through “Generate deeper family insight.”

## No-Raw-Audio-Storage Policy

Raw audio is not stored by default.

Supported input modes:

- `live_audio`: browser audio chunks are sent to speech-to-text and discarded after transcript generation.
- `uploaded_audio_transient`: uploaded files are used only for immediate transcription and then discarded.
- `transcript_upload`: pasted or uploaded transcripts skip speech-to-text cost.
- `manual_text`: manually entered conversation text is normalized into turns.

The app stores only transcript turns, speaker labels, timestamps, language, analysis nodes, recommendations, risk assessment, consent records, audit logs, and therapist notes. It does not expose raw audio URLs. `AudioProcessingEvent.audioPersisted=false` is logged when `STORE_RAW_AUDIO=false`.

If raw storage is explicitly enabled later with `STORE_RAW_AUDIO=true`, storage must be temporary and covered by Cloud Storage lifecycle deletion under `raw-audio/`.

## Redis Vs Firestore Choice

Redis/Memorystore is disabled for the MVP unless `ENABLE_REDIS=true`.

Reason: Memorystore is provisioned infrastructure and can add monthly cost. Firestore plus Cloud Run scale-to-zero is enough for MVP session state, job status, cached AI outputs, rate counters, transcript storage, and audit logs.

Local development uses the in-memory adapter when credentials are missing. Production defaults to Firestore unless Redis is explicitly enabled.

## Cost Principles

- Use Cloud Run scale-to-zero.
- Use Firestore for MVP session data, job status, cache, rate counters, transcript storage, and audit logs.
- Keep Redis/Memorystore disabled unless `ENABLE_REDIS=true`.
- Use Cloud Storage lifecycle deletion only if raw-audio storage is explicitly enabled.
- Do not store raw audio by default; process audio transiently and discard it after transcription.
- Prefer transcript upload when possible.
- Do not call AI on page load.
- Use rule-based analysis before Gemini.
- Cache AI outputs.
- Keep BigQuery, Cloud SQL, and real-time audio streaming out of MVP unless needed.
- Do not keep raw audio indefinitely.
- Do not keep duplicate transcripts.
- Compress summaries.
- Avoid BigQuery and long-term logs in MVP.

## Safety Principles

- Do not diagnose parent or child.
- Use coaching language.
- Require consent.
- Show professional review for high-risk content.
- Avoid hidden recording or child surveillance.
- No hidden recording.
- No ads based on child behaviour.
- No selling or sharing child behavioural data.
